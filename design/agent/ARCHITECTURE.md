## Agent 核心设计

### Agent 定义

Agent 是数字员工，与 t_employees（人类员工）并列，共同构成企业员工体系。

**核心属性**:
- `name/e_name`: 中英文名称
- `mode`: 服务模式（exclusive/shared/public/department）
- `config`: 模型配置 + 运行时配置 + 多模型协作配置
- `profile`: 个人事实与偏好
- `heartbeat`: 心跳配置

**运行时状态**（内存中）:
```typescript
interface AgentRuntime {
  // 基础信息
  sid: string;
  name: string;
  config: AgentConfig;
  
  // 运行时状态（不持久化）
  runtime: {
    status: 'idle' | 'running' | 'error';
    lastHeartbeat: Date;
    nextHeartbeat: Date;
  };
  
  // 内存缓存
  cache: {
    skills: Skill[];
    contacts: Map<string, ContactContext>;
  };
}
```

### 多模型协作配置

Agent 支持配置多模型协作模式，用于处理复杂的语音交互场景：

```typescript
interface MultiModelCollaborationConfig {
  /** 是否启用多模型协作 */
  enabled: boolean;
  /** 语音识别模型实例ID */
  speechRecognitionInstanceId?: string;
  /** 主对话模型实例ID */
  mainModelInstanceId?: string;
  /** 语音合成模型实例ID */
  speechSynthesisInstanceId?: string;
  /** 是否启用流式协作（实验性） */
  enableStreaming?: boolean;
}
```

**协作流程**:
1. **语音输入**: 语音识别模型(STT) → 文本
2. **主对话**: 主模型处理文本 → 生成回复
3. **语音输出**: 语音合成模型(TTS) → 音频

### Agent 与员工的关系建立

员工通过 **WebUI** 访问 Agent，关系建立流程：

```
员工登录 WebUI
    ↓
认证：t_users → t_roles → t_permissions
    ↓
识别身份：t_employees → t_contacts (type='employee')
    ↓
建立通道身份：r_channel_contact (cradle:user-id)
    ↓
访问 Agent：通过 t_relationship 验证关系
```
