# Cradle 系统设计文档

## 项目概述

Cradle 是一个面向企业的**私有化部署 AI 助理平台**，旨在为每个员工提供一对一的 Agent，配合完成工作、校对工作目标，并沉淀岗位工作逻辑，最终形成企业的数字员工资产。

## 核心目标

- 为每个员工提供专属 Agent，提升个人工作效率
- 为每个部门提供对外服务窗口，促进跨部门协作
- 沉淀企业知识和岗位经验，形成可复用的数字资产
- 支持业务系统数据查询，打通信息孤岛

## 与 OpenClaw 的关系

Cradle 基于 OpenClaw 进行解构和重新设计：

- **保留优点**: Skills 系统、多通道接入、Agent 运行时架构
- **改进方向**: 企业级权限管理、多用户支持、数据资产沉淀、WebUI 优先

## 架构原则

### 设计原则

1. **简单优先**: MVP 阶段以快速实现为核心目标
2. **扩展预留**: 架构设计预留未来扩展空间（如 MySQL 迁移、多模型路由）
3. **兼容并包**: Skills 系统兼容标准规范，便于生态对接
4. **数据资产**: 将 Agent 记忆和工作逻辑视为企业核心数据资产

### 技术选型

| 层级 | 技术 | 说明 |
|-----|------|------|
| 后端语言 | TypeScript | 沿用 OpenClaw 技术栈，便于迁移 |
| 运行时 | Node.js ≥22 | 与 OpenClaw 保持一致 |
| 数据库 | SQLite | MVP 阶段，未来迁移至 MySQL |
| 向量存储 | SQLite + sqlite-vec | 轻量级向量存储 |
| 前端 | React/Vue + TypeScript | WebUI 界面 |
| 通信 | WebSocket | 实时对话和状态推送 |
