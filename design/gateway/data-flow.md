## 组件职责

### Master 进程职责

| 组件 | 职责 | 关键方法 |
|------|------|----------|
| HTTP Server | 接收外部请求 | `listen()`, `on('request')` |
| WebSocket Handler | 管理 WebSocket 连接 | `handleUpgrade()`, `on('connection')` |
| Channel Manager | 管理 Channel 插件 | `loadChannel()`, `getChannel()` |
| Message Queue | 消息缓冲与分发 | `push()`, `pop()` |
| Worker Manager | 管理 Worker 进程 | `forkWorker()`, `sendToWorker()` |
| Gateway Router | 消息路由 | `route()`, `identifyContact()` |

### Worker 进程职责

| 组件 | 职责 | 关键方法 |
|------|------|----------|
| Message Consumer | 消费队列消息 | `consume()`, `processMessage()` |
| Identity Service | 身份识别 | `identifyContact()`, `identifyAgent()` |
| Routing Service | 路由决策 | `resolveRoute()`, `checkPermission()` |
| Agent Runtime | Agent 执行 | `process()`, `callLLM()` |
| Channel Adapter | 出站消息发送 | `format()`, `send()` |

### Channel 插件职责

| 方法 | 调用时机 | 功能 |
|------|----------|------|
| `parse()` | Master 接收消息后 | 解析平台特定格式 |
| `normalize()` | Master 解析后 | 转为统一格式 |
| `format()` | Worker 发送前 | 转为平台格式 |
| `send()` | Worker 格式化后 | 调用平台 API |

## 数据格式

### InboundMessageContext (入站消息上下文)

```typescript
interface InboundMessageContext {
  // 消息标识
  messageId: string;           // 全局唯一消息ID
  
  // 通道信息
  channelType: string;         // 'webui' | 'dingtalk' | 'wechat' | ...
  channelId: string;           // 通道实例ID
  
  // 发送者信息
  senderId: string;            // 发送者在通道中的ID
  senderName?: string;         // 发送者名称
  
  // 接收者信息
  recipientId: string;         // 接收者在通道中的ID
  recipientName?: string;      // 接收者名称
  
  // 消息内容
  body: string;                // 消息正文
  contentType?: string;        // 'text' | 'image' | 'file' | ...
  attachments?: Attachment[];  // 附件列表
  
  // 会话信息
  chatType: 'direct' | 'group' | 'channel';  // 会话类型
  chatId: string;              // 会话ID
  
  // 时间戳
  timestamp: number;           // 消息时间戳
  
  // 原始数据
  raw: unknown;                // 原始平台消息
}
```

### OutboundMessageContext (出站消息上下文)

```typescript
interface OutboundMessageContext {
  // 消息标识
  messageId: string;           // 回复消息ID
  replyTo?: string;            // 回复的目标消息ID
  
  // 通道信息
  channelType: string;         // 目标通道类型
  channelId: string;           // 目标通道ID
  
  // 接收者信息
  recipientId: string;         // 接收者在通道中的ID
  
  // 消息内容
  body: string;                // 消息正文
  contentType?: string;        // 'text' | 'image' | 'file' | ...
  attachments?: Attachment[];  // 附件列表
  
  // 会话信息
  chatType: 'direct' | 'group' | 'channel';
  chatId: string;              // 会话ID
  
  // 时间戳
  timestamp: number;
}
```

### ChannelMessage (通道间通信消息)

```typescript
interface ChannelMessage {
  // 消息类型
  type: 'inbound' | 'outbound' | 'system';
  
  // 消息ID
  id: string;
  
  // 消息上下文
  context: InboundMessageContext | OutboundMessageContext;
  
  // 元数据
  metadata: {
    channelId: string;
    workerId?: string;
    processingTime?: number;
    retryCount: number;
  };
  
  // 状态
  status: 'pending' | 'processing' | 'completed' | 'failed';
  
  // 错误信息
  error?: {
    code: string;
    message: string;
    stack?: string;
  };
}
```

## 错误处理

### 错误分类

| 错误类型 | 发生阶段 | 处理方式 |
|----------|----------|----------|
| 解析错误 | Channel.parse() | 记录日志，返回错误响应 |
| 认证错误 | HTTP Server | 返回 401/403，断开连接 |
| 路由错误 | Gateway Router | 记录日志，返回错误消息 |
| 处理错误 | Worker | 重试3次，失败后入死信队列 |
| 发送错误 | Channel.send() | 重试3次，失败后通知管理员 |

### 重试策略

```typescript
interface RetryPolicy {
  maxRetries: number;          // 最大重试次数 (默认3)
  backoffStrategy: 'fixed' | 'exponential';  // 退避策略
  initialDelay: number;        // 初始延迟 (ms)
  maxDelay: number;            // 最大延迟 (ms)
}

// 指数退避示例
const delay = Math.min(
  initialDelay * Math.pow(2, retryCount),
  maxDelay
);
```
